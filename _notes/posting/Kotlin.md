# Kotlin

```kotlin
val / var  
→ 불변/가변 변수 선언. 함수형 스타일 코딩의 기초

fun  
→ 함수 선언 키워드

Unit  
→ Kotlin의 void 개념 (함수 반환 타입)

null safety ( ?, ?:, !! )  
→ Null 안정성 – Kotlin의 대표 특징

data class  
→ 불변 객체 + 자동 equals/hashCode/toString 생성

object / companion object  
→ 싱글톤 및 static 대체용 키워드

when  
→ Kotlin의 switch 대체. 더 표현력 좋고 강력함

extension function  
→ 기존 클래스에 함수 추가 가능. 선언적 코딩 가능

lambda / higher-order function  
→ 함수형 프로그래밍 핵심. 함수를 값처럼 사용

coroutines (suspend / launch / async)  
→ 비동기 처리의 핵심. 경량 스레드 기반

sealed class  
→ 제한된 상속 계층. when과 함께 패턴 매칭에 적합

inline / reified  
→ 성능 최적화와 제네릭 타입 보존을 위한 고급 키워드
```
